Repositories: Centralized folders that host your project's files, documentation, and the complete version history of every change.

Branches: Isolated environments that allow you to develop features or fix bugs without affecting the stable "main" codebase.

Pull Requests: A formal way to propose changes, allowing team members to review code and discuss improvements before merging.

GitHub Actions: An automation engine that handles CI/CD workflows, such as automatically testing and deploying code to the cloud.

Issues: A built-in tracking system to report bugs, request new features, and manage project tasks with labels and milestones.

GitHub Copilot: An AI-powered coding assistant that provides real-time suggestions for functions and logic directly in your editor.

Codespaces: Instant, cloud-hosted development environments that allow you to start coding in a browser without local setup.

Dependabot: An automated security tool that scans your dependencies for vulnerabilities and creates pull requests to update them.
